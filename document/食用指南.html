移植

本节以"..\library\empty"文件夹下的空白程序为模板，只说明程序的移植，工程文件的添加自行搜索😄。

文件添加

完整的程序有如下文件：

  • 第一、第二个文件为Cotex-M4的FPU文件，如果用的是其他内核的就不用复制，Cotex-M7也有FPU但没试过。
  • 第三个"IIR_fdacoefs.h"和最后一个"tmwtypes.h"为MATLAB提供的IIR滤波器文件，前者的滤波数据需要使用MATLAB的fdatool工具生成，后者一般在MATLAB的安装目录中（第一个文件生成时里面会有说明地址在哪，也可以使用本项目的但不保证没问题）。
  • 倒数第二、第三的“Smooth_Filter.c/h”为平均滑动滤波器，如果不需要磁力计（其他数据默认使用IIR滤波器）可以不添加。

程序修改

除了提到的部分，其他代码都不建议修改。

Matrix.c/h

此文件为矩阵运算文件，默认使用.c文件中的软件计算，如果使用的是Cotex-M4内核的话可以使用FPU进行计算。

    //1：Cotex-M4硬件解算

注意，如果使用硬件解算还需要添加以下宏定义：

,ARM_MATH_CM4, __FPU_PRESENT

具体可以参考例程的《快速移植说明》。

IIR_fdacoefs.h

把MATLAB生成的滤波器参数文件全部复制进来就行，具体生成步骤可以看“滤波”一节。

IIR_Filter.c/h

一般不用改，如果编译时报错“The buffer is too small！”说明缓存数组长度不够，加大头文件中下面宏定义的数值即可。

xxxxxxxxxx
#define IIR_SECTIONS_BUF    10

内存够的话固定在一个较大的值即可，这样写只是为了方便滤波器参数配置。

如果内存吃紧可以自己计算一下，数值为生成的IIR滤波器节数，即阶数除以2。

Smooth_Filter.c/h

同上，一般不用改，默认滤波器最大深度为8，如果觉得滤波效果不行需要使用更多的数据，增大下面的宏定义即可。

xxxxxxxxxx
#define SMOOTH_FILTER_DATA_DP  8

为了方便说明，用空行分为了上下两部分。

第一部分涉及解算数据，看情况可以适当修改，但一般保持默认：

  • 前两行为Mahony算法参数的调节（类似PID），会影响解算数据的响应（具体某度搜索），数据响应异常可以尝试调整但不建议。
  • 接着的三行为角加速度死区范围，理论上为0最好（没有死区），但噪声对数据漂移影响较大（特别是yaw轴），可以适当调整，参考“滤波”→“死区控制”一节。
  • 第一部分最后一行为重力加速度，如果获取到当地的重力加速度可以改为对应的数值。

第二部分为程序设定，按照使用的平台进行修改：

  • 前两行为延时函数，考虑到系统的使用分为了两个，第一个延时只在初始化时使用，第二个延时只在工作时（在任务中）使用。例如使用FreeRTOS，第二个就应改为vTaskDelay(n)。
  • 第三行为预解算的次数，如果结算程序已经开始了但数值还在变化（陀螺仪静止）随后稳定（没做四元数初值计算，直接用解算来初始化四元数），可适当增大数值。
  • 最后两行分别为磁力计使能和加速度校准使能，前者为0即可；后者视情况决定，如果没有存储空间（校准数据需要存储）或没有校准数据可以置0关闭校准。

校准

角加速度校准

角加速度的校准比较简单，在IMU_Init_Calibration_Gyro()中完成，每次程序进行初始化时都会进行一次校准数据的获取，故初始化时必须保持陀螺仪的静止。

加速度校准

加速度校准相对复杂（其实也不难😄），以默认程序为例和Keil，需要进入Debug模式打断点进行校准，校准程序如下：

xxxxxxxxxx
void IMU_Get_Accel_Cali_Data(IMU_Data_t *imu_data) {
    static int Cali_Sta = 0, Pos_N = 0;
    while(!Cali_Sta) {
        if((Pos_N > 0) && (Pos_N <= 6)) {
            IMU_Calibration_Accel_Get_Raw(imu_data);
            imu_data->imu_calibration_data.Accel_W_raw_data[Pos_N-1][0] = imu_data->imu_calibration_data.accel_x;
            imu_data->imu_calibration_data.Accel_W_raw_data[Pos_N-1][1] = imu_data->imu_calibration_data.accel_y;
            imu_data->imu_calibration_data.Accel_W_raw_data[Pos_N-1][2] = imu_data->imu_calibration_data.accel_z;
            imu_data->imu_calibration_data.Accel_W_raw_data[Pos_N-1][3] = 1;
            Pos_N = 0;
        }
        else if(Pos_N == 7) {
            IMU_Calibration_Accel_Calc_CaliMat(imu_data);
            Pos_N = 0;
        }
        IMU_Task_Delay_ms(1);
    }
}

校准步骤如下：

  1. 在任务中调用此函数进入循环；

  2. 将"Cali_Sta"、"Pos_N"两个变量加入watch窗口；

  3. 在"if"处打断点，程序停止于此时修改变量"Pos_N"的数值获取对应面的加速度。

    数值的有效范围为1到7：

    参考编号1的正确方向如下：

     

    校准方法为“六面校准法”，即将陀螺仪当做一个标准的正方体平放，用重力进行校准，校准时注意以下几点：

    • 校准时应确保陀螺仪放置于水平面上，倾斜面将导致数据不可用！
    • 校准时需要获取一定数量的原始数据，应保持陀螺仪静止；
    • 暂存校准数据的"Accel_W_raw_data"数组默认初值均为0，校准时要一次6面都进行校准，否则校准数据将不可用或矩阵计算错误，会卡死在IMU_Calibration_Accel_Calc_CaliMat()函数的循环中（例如矩阵求逆）。

  4. 获取完六面数据后将变量"Pos_N"改为7存储校准数据，退出校准循环则将变量"Cali_Sta"置1。

该函数为外部API，如果不需要在Debug模式下进行校准可按原程序校准流程和要求修改此函数。

一般只要陀螺仪和对应的单片机不更换，加速度校准只做一次并储存即可，每次上电程序都会获取校准数据。

注意，即时不使能加速度校准也一样会使用矩阵运算。

死区控制

死区控制主要是为了减缓Yaw的漂移，但这是牺牲了动态精度换取的静态稳定性，如果在校准或其他方法都已优化仍有较快的漂移的话，可以适当增加死区，建议在默认值（0.002）上下。注意这个值是经过单位换算后的弧度制的数据。

记住这是抑制Yaw轴漂移的最后手段，六轴的Yaw轴漂移是一定存在的，算法、滤波都只能减缓而不能消除，死区调整至一个能接受的数值（一分钟零点几度那样）就差不多了。
